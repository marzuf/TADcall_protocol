Help for CARPnTrees web application

From ancient and recent polymorphism phylogenies to a consensus tree

This application can be used to build consensus tree from ancient and recent polymorphism phylogenies, as well as these latter ones. The ancient polymorphism tree (haplogroup tree) gives the topology of the basis of the tree. Then, recent polymorphism phylogeny is built for each subset of individuals that share the same haplogroup. These "mini-trees" are appended at the corresponding tip of the haplogroup tree, resulting in a consensus tree (the methodology in details).

Input data

Recent polymorphism

  • For analysis of mtDNA genetic data, input data should be aligned sequences in FASTA format.
  • For analysis of Y-chromosome genetic data, input data should be a data frame containing microsatellites data (individuals in rows, loci in columns; individual ID in the first column/row names).

Ancient polymorphism

Ancient polymorphism data consist in the haplogroup of every individuals. They must be provided in a table with individuals in rows, their ID as row names (first column) and their corresponding haplogroup in the first column after row names. For paternal lineages, haplogroup data can be in the same file as the microsatellites data (first column after row names).

Haplogroup tree

The inputed haplogroup tree for mtDNA or Y-chromosome must be in Newick format. If you do not have one, we provide one for maternal lineage (published here). Unfortunately, the haplogroup tree for paternal lineage is not yet available.

Output

The outputed trees can be downloaded in Newick format and the plots can be saved in png format.

Source files

The CARPnTrees program can also be launched from command line and is available in R and Python versions. Further explanations and all source files are available in our git repository .
